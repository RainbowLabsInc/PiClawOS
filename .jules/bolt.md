## 2024-05-24 - [CPU Temperature Reading Refactor]
**Learning:** Identifying redundant ad-hoc logic for common operations like temperature checks across the codebase can yield significant performance boosts and simplify maintenance. Specifically, the manual subprocess call to `vcgencmd` blocked and introduced unnecessary latency when an optimized, dedicated utility (`current_temp` in `pi_info`) already existed. The fallback (`psutil.sensors_temperatures()`) does extensive recursive system I/O which heavily bogs down devices like the Raspberry Pi. Unifying these calls under a fast sysfs read ensures both performance and consistency.
**Action:** Always scan the codebase for centralized utility functions before implementing repetitive systems-level I/O tasks. Prioritize lightweight sysfs file reads (`/sys/class/...`) over spawning heavy subprocesses (`vcgencmd`) or utilizing 'kitchen sink' fallbacks (`psutil.sensors_temperatures`) on constrained hardware. Ensure robust and correct `patch` placement in tests when substituting logic across modules.
