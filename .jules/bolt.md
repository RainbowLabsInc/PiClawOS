
## 2025-02-28 - Combining Multiple String Regex Subs into a Single Optimized Regex
**Learning:** In the `marketplace.py` tool, there was an array of "noise words" that were compiled into separate regex patterns. A loop iterated through these patterns, running `pattern.sub(" ", query)` for every single word. This resulted in significant overhead from matching multiple regexes separately and allocating intermediate strings. By combining all noise words into a single optimized regex using an OR `|` operator and formatting them into a single `re.compile(r"(?i)(?:^|(?<=\W))(?:(" + "|".join(...) + r"))(?:(?=\W)|$)")`, the runtime for query cleaning reduced from ~2.53 seconds to ~0.19 seconds per 10,000 runs (~13.7x speedup). Sorting keywords by length descending before combining avoids eager partial matching.
**Action:** When filtering or cleaning strings with arrays of predefined keywords, always combine them into a single compiled regex rather than iterating over individual compiled patterns.
