## 2026-04-15 - Prevent Argument Injection in Subprocess Calls
**Vulnerability:** External tools like `nmap` and `ping` called via `asyncio.create_subprocess_exec` received raw string arguments derived from user input (e.g., `ip` or `ip_range`). Because `nmap` interprets strings starting with `-` as command flags, attackers could inject arbitrary arguments (like `-iL /etc/passwd` or `-oG`), reading arbitrary files or altering execution flow (Argument Injection).
**Learning:** `asyncio.create_subprocess_exec` prevents shell injection (like `; rm -rf /`) because it bypasses the shell, but it does NOT protect against Argument Injection if the target executable natively parses its positional arguments as flags.
**Prevention:** Strictly validate all arguments passed to subprocess tools against an explicit format. For networking tools, use `ipaddress` for strict IP/CIDR validation and a tight regex (`^[a-zA-Z0-9.-]+$`) for hostnames. Explicitly reject any input starting with `-` before handing it to the executable.
