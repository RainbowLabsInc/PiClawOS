## 2024-05-18 - Fix Command Injection in Network Tool
**Vulnerability:** The `wifi_connect` and `wifi_disconnect` functions in `piclaw-os/piclaw/tools/network.py` used string interpolation (f-strings) to construct shell commands for `nmcli` and executed them using `asyncio.create_subprocess_shell`. This created a critical command injection vulnerability where a malicious SSID or password could include shell metacharacters (e.g., `"; rm -rf /"`) to execute arbitrary code.
**Learning:** Even internal utility functions that construct shell commands need rigorous protection against injection. Passing unsanitized user inputs to a shell is inherently unsafe. A previous attempt to fix the general `shell` tool by removing shell evaluation entirely broke expected shell functionality (like pipes and redirection), highlighting that fixes must preserve the tool's intended capabilities.
**Prevention:** To prevent command injection while preserving functionality, replace `asyncio.create_subprocess_shell` with `asyncio.create_subprocess_exec` when executing specific binaries (like `nmcli`). Pass arguments securely as elements of a `list[str]` rather than a single interpolated string. This ensures the operating system treats the arguments strictly as data, bypassing shell evaluation.
